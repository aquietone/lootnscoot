Settings Reference

Overview

LootNScoot stores per-character settings as a Lua table serialized to disk using mq.pickle(). mq.pickle() is a MacroQuest (MQ) function that converts a Lua table into a human-readable text file and writes it to disk. The output is valid Lua code, so you can open the file in a text editor and read it directly. This is why it is preferred over binary formats — easy to inspect and manually edit if needed. The settings file path follows the MQ convention: 

config/LootNScoot/[ServerName]/[CharName].lua

When the script starts, it loads settings from this file. If the file does not exist, default values from modules/settings.lua are used and a new file is created. Settings are saved automatically when changed through the UI or CLI commands, calling LNS.writeSettings() which serializes the table back to disk with mq.pickle().

The settings module is defined in modules/settings.lua and accessed throughout the codebase as settings.Settings.<SettingName>.

Settings by Category

Looting Behavior

These settings control what gets looted and how the looting process works.

SettingTypeDefaultDescription
DoLootbooltrueEnable automatic looting in standalone mode. When false, the script runs but does not loot corpses.
LootForagebooltrueEnable looting of foraged items (items your character picks up from the ground using the Forage skill).
LootNoDropboolfalseEnable looting of NoDrop items. NoDrop is an EverQuest item flag meaning the item cannot be traded to other players once picked up — it is permanently bound to the character who loots it.
LootNoDropNewboolfalseEnable looting of NoDrop items that have no existing rule. Both LootNoDropNew and LootNoDrop must be enabled for new NoDrop items to be added to the database.
LootQuestbooltrueEnable looting of items marked with a "Quest" rule. NoDrop quest items also require LootNoDrop to be on.
LootMyCorpseboolfalseLoot your own character's corpse if it is nearby. Does not check for pending resurrections.
LootAugmentsboolfalseLoot augmentation items. Overrides global rules for augments; personal rules still take priority.
KeepSpellsbooltrueKeep spell scrolls regardless of global rule. Personal rules will override this setting.
CanWearboolfalseOnly loot NoDrop new items that your character's class can use. This controls whether NoDrop items are looted (picked up from the corpse into inventory), not whether they are equipped. The EQ game client may auto-equip looted items into empty equipment slots.
CheckCorpseOncebooltrueOnly check each corpse once. If enabled, previously looted corpses are skipped on subsequent scans.
StackableOnlyboolfalseOnly loot stackable items (items that can stack in a single inventory slot).
CombatLootingboolfalseAllow looting while in combat. Not recommended for the main tank.
MaxCorpsesPerCyclenumber5Maximum number of corpses to process per lootMobs() call. Controls throughput vs. time spent looting.
LootCheckDelaynumber0Seconds to wait between loot checks. Also applied to locked corpses before rechecking them.
NavTimeoutnumber5Maximum seconds to spend navigating to a corpse before giving up.

Radius and Range

These settings define the distances (in game units) used for corpse scanning.

SettingTypeDefaultDescription
CorpseRadiusnumber100Horizontal radius to scan for lootable corpses.
CorpseZRadiusnumber50Vertical (Z-axis) radius to scan for corpses. Prevents looting corpses on different floors.
MobsTooClosenumber40If hostile NPCs are within this range (plus CorpseRadius), looting is skipped unless CombatLooting is on.

Inventory

Settings that control bag and inventory management.

SettingTypeDefaultDescription
SaveBagSlotsnumber3Number of bag slots to keep empty at all times. Looting stops when free slots drop to this number.
IgnoreBagSlotnumber0Skip this bag slot number (1-10) when buying, selling, tributing, and destroying items. 0 means no slot is ignored.

Destroy

Settings for the destroy (permanently delete) functionality.

SettingTypeDefaultDescription
DoDestroyboolfalseEnable destroy functionality. When false, items marked "Destroy" are treated as "Ignore" instead. This is a safety setting.
AlwaysDestroyboolfalseDestroy all non-quest items marked "Ignore" to clean corpses. Requires DoDestroy to also be true.

SettingsEnum

TempSettings

TempSettings is a table of runtime-only volatile state that is never saved to disk. It is reset every time the script starts. This separation from Settings is intentional: Settings holds persistent configuration that survives restarts (saved via mq.pickle()), while TempSettings holds transient UI state, inter-frame communication flags, and temporary data that only makes sense during the current session. Mixing them would clutter the settings file with values that are meaningless on reload.

BuyItemsTable

The BuyItemsTable is stored inside the settings and persisted to disk. It is a simple Lua table mapping item names to desired quantities: 

{
    ["Bone Chips"] = 100,
    ["Bat Wing"] = 20,
    ["Fish Rolls"] = 40,
}

Items in this table are automatically purchased from vendors after selling (when AutoRestock is true) or on demand via /lns restock. See the CLI Commands page for how to manage buy items.

To add via CLI: /lns buy "Bone Chips" 100
To add with item on cursor: /lns buy (adds 1) or /lns buy 100 (adds 100)

Tip: Use /lns config to print all current setting values to the MQ console. This is the quickest way to verify your configuration without opening the UI.
