User Interface

Overview

LootNScoot's entire graphical interface is built with ImGui (Immediate Mode GUI), a UI library embedded within MacroQuest (MQ). If you have never worked with ImGui before, the key concept is this: the UI is redrawn from scratch every single frame. There is no persistent widget tree or stored state in the UI layer itself. Every frame, your rendering function runs, issues drawing commands (ImGui.Text(), ImGui.Button(), etc.), and ImGui turns those commands into pixels. This is called immediate-mode rendering.

The UI is registered during initialization in init.lua with a single call: 

mq.imgui.init('LootnScoot', ui.RenderUIs)

mq.imgui.init() tells MacroQuest to call ui.RenderUIs() once per frame. That one function is the entry point for the entire UI: it orchestrates every window, popup, and overlay that LootNScoot displays. The UI module lives in modules/ui.lua.

Main Window

The primary window is titled "LootnScoot" and is rendered by LNS_UI.RenderMainUI(). It has two major display modes: Items Mode and Settings Mode, toggled by a button in the toolbar. A small floating button widget (RenderBtn()) is also always visible and provides quick access to toggle the main window.

Toolbar

Across the top of the main window you will find a row of toolbar controls:

ControlPurpose

Items Mode

When in Items mode the function drawItemsTables() renders a tab bar with three tabs:

  • Global_Rules — rules that apply to all characters on the server.
  • [YourName]_Rules — personal rules specific to the current character.
  • All Items — every item LootNScoot has ever seen, pulled from the SQLite database.

Each tab displays a paginated, searchable, sortable table. Pagination defaults to 25 items per page (the constant ITEMS_PER_PAGE) but you can change it from a dropdown (10 to 100 in increments of 10). Navigation controls include first-page, previous, next, and last-page buttons, plus a page counter.

Every item row shows an icon, name, sell value, tribute value, and many other columns (stackable, NoDrop, Lore, tradeskill, classes, stats, resists, heroics, and more). Most columns are hidden by default and can be revealed via ImGui's built-in column visibility menu (right-click a column header).

Interaction on an item row:

  • Left-click the icon — opens the EverQuest in-game item details window via /executelink.
  • Right-click the icon — pops open a standalone details window rendered by Draw_item_info_window().
  • Click the item name (Selectable) — opens the Modify Item window for that item.

The Items mode also includes a BulkSet collapsible section. This lets you select multiple items with checkboxes, choose a rule and class restriction, and apply the rule to all selected items at once. The bulk operation broadcasts rule changes via the Actor system (MQ's inter-process messaging) to other running characters.

When new items are pending (items found on corpses that have no rule yet), a New Items tab appears automatically.

Settings Mode

When switched to Settings mode, renderSettingsSection() draws the settings editor. It contains:

  • Actor Selector Dropdown — a combo box listing all connected characters (populated from LNS.BoxKeys). Selecting a different character lets you view and modify their settings remotely. Changes are broadcast via the updatesettings actor action.
  • Boolean Toggles — each boolean setting is rendered as a toggle switch. The list of settings to display is determined by filtering settings.Settings for boolean-type values and excluding anything in settings.SettingsNoDraw.
  • Numeric Inputs — settings like CorpseRadius, SaveBagSlots, MinSellPrice, etc. are rendered as number input fields.
  • Channel Dropdowns — LootChannel and GroupChannel are rendered as combo boxes or text inputs for selecting the in-game chat channel (e.g., dgt, dgze, say).

New Item Popup

Rendered by renderNewItem(), this window appears when a new item is discovered on a corpse and no rule exists for it yet. If the setting AutoShowNewItem is enabled, it pops up automatically; otherwise the user can open it from the New Items tab.

The popup displays:

  • Item icon (rendered via drawIcon()).
  • Item name, ID, and item link.
  • Sell price (broken down into platinum, gold, silver, copper via draw_value()), tribute value, and max stack size.
  • Class restrictions, NoDrop (cannot be traded), Lore (only one can be held per character), Tradeskill, and Augment flags.
  • Corpse ID the item was found on.

Below the item details is a row of radio buttons for selecting a loot rule. The available choices come from the settingList array: 

local settingList = {
    "Ask", "CanUse", "Keep", "Ignore", "Destroy",
    "Quest", "Sell", "Tribute", "Bank",
}

Additional controls in the popup:

  • "Loot All" checkbox — if checked, the class restriction is ignored and anyone can loot the item.
  • Classes input — a text field to restrict which character classes may loot the item (e.g., shm clr dru).
  • "Share to Others" checkbox — when checked, the rule decision is broadcast to all other connected characters via the Actor system.
  • OK button — saves the rule and removes the item from the new items queue.
  • Cancel button — dismisses without saving.
  • Delete button — removes the item entry entirely.
  • Inspect (icon click) — left-click the icon to open the in-game item window, right-click to pop open a details overlay.

Modify Item Window

Rendered by RenderModifyItemWindow(), this window opens when you click an item name in any items table, or when you use the /lns command to modify an item by name. It allows editing the rule for a specific item.

Features of the Modify Item window:

  • Item Search — if the item ID is negative (a legacy ID from the old INI system), LootNScoot automatically searches the database for matching items by name and displays a match table.
  • Rule Radio Buttons — the same settingList radio buttons as the New Item popup, plus a Quest quantity input when Quest is selected.
  • Class Restriction Editing — a text input field for specifying allowed classes (e.g., shm clr dru or All).
  • Table Selector — a dropdown to choose whether the rule goes into Global_Rules or [YourName]_Rules (personal).
  • "Share to Other Characters" checkbox — for personal rules, broadcasts the change to other logged-in characters via sharePersonalRuleToOthers().
  • Item Link Display — shows the EQ item link; clicking it opens the in-game item window.
  • Set Rule button — commits the change by calling setGlobalItem() or setPersonalItem().
  • Trash button — deletes the rule entirely.
  • Cancel button — closes without saving.

When a name search produces multiple matches, a sub-table is displayed with columns for item ID, item name, and a Set button. Clicking Set on a match replaces the old (legacy) entry with the correct database entry.

Master Looter Window

Rendered by RenderMasterLooterWindow(), this window only appears when LNS.MasterLootList contains data (i.e., when Master Looter mode is active and items have been found on corpses). See the Multi-Character page for how the Master Looter system works at the messaging level.

The window shows a list of corpses, each with its items. For each item, you see:

  • The item name and link.
  • Item value, NoDrop status, and Lore status.
  • A Members table showing each connected character and how many of that item they already own (inventory + bank count).
  • "Loot for [CharName]" buttons for each member, which send a loot_item actor message to the target character.
  • Recheck button to re-broadcast a recheck_item message and refresh member counts.

Console Window

The console is powered by the guiLoot module (modules/loot_hist.lua). It is a scrollable, filterable log of all loot events, actor messages, debug output, and errors. Key features:

  • Scrollable text output with automatic scroll-to-bottom.
  • Search and filter functionality to find specific loot events.
  • Theme selection — the console supports the same theme system as the rest of the UI.
  • Import GUI elements — LootNScoot injects a custom "Loot N Scoot" menu into the console's menu bar via guiExport(), providing quick access to Sell, Restock, Tribute, Bank, and Cleanup commands for both the local character and the group.

Report Windows

The report system consists of two related views, both rendered by DrawRecord():

  • Session Report — shows all loot events from the current play session, stored in settings.TempSettings.SessionHistory.
  • Historical Data Lookup — opened via the Past History window. A date selector dropdown (populated from LNS.HistoricalDates) lets you load loot records for a specific date. You can also search by item name. The table displays Date, Timestamp, Item, Looter, Action, Corpse Name, and Zone columns, with pagination and filtering.

Help Window

Rendered by renderHelpWindow(), this is a built-in command reference displayed in a standalone ImGui window. It contains collapsible sections with tables listing:

  • Run Commands — how to launch LootNScoot in different modes (standalone, once, directed).
  • Basic Commands — /lns sellstuff, /lns bankstuff, /lns tributestuff, /lns loot, /lns set, /lns pause, /lns resume, /lns corpsereset, /lns help.
  • Item Commands — /lns [sell|keep|quest|bank|ignore|destroy], /lns buy, /lns find, /lns personalitem, /lns globalitem.
  • Toggles — a dynamically generated list of all boolean settings with their tooltip descriptions.
  • Settings — a dynamically generated list of all non-boolean settings with their tooltips.

Other Windows

Several additional windows are rendered by RenderUIs() when their conditions are met:

  • Clone Window (renderCloneWindow()) — allows copying settings from one character to another. Useful when setting up a new alt.
  • Import DB Window (renderImportDBWindow()) — provides a UI for importing an external items database file. Only shown when settings.TempSettings.ShowImportDB is true.
  • Performance Monitor — rendered by the performance module (modules/performance.lua), it uses ImPlot (a plotting extension for ImGui) to display real-time frame timing graphs. Useful for diagnosing UI performance issues.
  • Item Detail Popups (Draw_item_info_window()) — standalone windows showing full item statistics. These pop open when you right-click an item icon anywhere in the UI. They display stats, resists, augment slots, damage info, and more, rendered with colored text via the Colors() function.

UI Architecture

Understanding how the pieces fit together will help you navigate the code: 

RenderUIs()                          [called every frame by mq.imgui]
  |
  +-- DrawTheme()                    [apply color/style theme]
  |
  +-- enterNewItemRuleInfo()         [process queued new-item decisions]
  |
  +-- RenderModifyItemWindow()       [if ModifyItemRule flag is set]
  |
  +-- renderNewItem()                [if NewItemsCount > 0]
  |     +-- Draw_item_info_window()  [for each right-click popup]
  |
  +-- processPendingItem()           [if pendingItemData exists]
  |
  +-- renderImportDBWindow()         [if ShowImportDB flag is set]
  |
  +-- RenderMainUI()                 [the primary LootnScoot window]
  |     +-- drawItemsTables()        [Items mode: tabs + paginated tables]
  |     +-- renderSettingsSection()  [Settings mode: toggles + inputs]
  |
  +-- RenderBtn()                    [floating mini-button widget]
  |
  +-- renderCloneWindow()            [clone settings between characters]
  |
  +-- RenderMasterLooterWindow()     [if MasterLootList has data]
  |
  +-- DebugMailBox()                 [if ShowMailbox is true]
  |
  +-- renderHelpWindow()             [help command reference]
  |
  +-- DrawRecord()                   [if PastHistory flag is set]
  |
  +-- perf:Render()                  [if performance monitor is visible]
  |
  +-- PopStyleColor / PopStyleVar    [clean up theme]

DrawTheme()

At the start of every frame, RenderUIs() calls LNS.guiLoot.DrawTheme(). This pushes ImGui style colors and vars onto the stack, returning counts of how many were pushed. At the end of the frame, those are popped to restore defaults. This is how the theme system works in ImGui: push overrides at the top, pop them at the bottom.

Color System

The function LNS_UI.Colors(color_name) returns an ImVec4 color value by looking up a name in the ColorList table. Available color names include: red, pink, orange, tangarine, yellow, white, blue, softblue, teal, green, grey, purple, and more. If a color name is not found, white (ImVec4(1,1,1,1)) is returned as a fallback.

Icon Rendering

EverQuest item icons are rendered using texture animation. The setup: 

local iconAnimation = mq.FindTextureAnimation('A_DragItem')
local EQ_ICON_OFFSET = 500

function LNS_UI.drawIcon(iconID, iconSize)
    if iconSize == nil then iconSize = 16 end
    if iconID ~= nil then
        iconAnimation:SetTextureCell(iconID - 500)
        ImGui.DrawTextureAnimation(iconAnimation, iconSize, iconSize)
    end
end

mq.FindTextureAnimation('A_DragItem') loads the EQ item icon sprite sheet. Each item has an icon ID; subtracting the offset of 500 gives the correct cell index in the texture atlas. DrawTextureAnimation() then renders that cell at the specified size.

Tip: Right-clicking an item icon anywhere in the UI will pop open a detailed item stats window. Left-clicking opens the EverQuest in-game item details window. This works in the Items tables, the New Item popup, and the All Items database view.
Warning: Never call mq.delay() from inside an ImGui render function. mq.delay() is an MQ function that pauses script execution for a specified duration (e.g., mq.delay(1000) pauses for 1 second). ImGui render callbacks run on the main thread every frame. Calling mq.delay() inside one will freeze the entire UI (and possibly the game client) for the duration of the delay. All time-dependent logic must be handled in the main loop, not the render loop. Use TempSettings flags to communicate between the render function and the main loop instead.
