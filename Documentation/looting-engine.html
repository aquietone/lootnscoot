Looting Engine

Overview

The looting engine is the core pipeline that drives LootNScoot. It scans for nearby corpses, navigates your character to each one, opens the loot window, evaluates every item against the rule system, and picks up (or destroys, or skips) each item accordingly. After looting, items sitting in your inventory can be sold, banked, tributed, destroyed, or restocked through the processItems() state machine.

EverQuest (EQ) is a 3D MMO where defeated enemies leave behind "corpses" containing loot. MacroQuest (MQ) is a third-party tool that embeds a Lua scripting engine and exposes game data through "TLO" (Top-Level Object) accessors—for example, mq.TLO.Me.FreeInventory() returns how many empty bag slots you have. LootNScoot automates the tedious process of right-clicking every corpse and deciding what to pick up.

Finding Corpses: lootMobs()

The function LNS.lootMobs(limit) is the top-level entry point. It is called on every loop tick in standalone mode, or on demand via /lns loot. The limit parameter defaults to 50 inside the function, but callers typically pass settings.Settings.MaxCorpsesPerCycle, capping how many corpses are processed per call.

Pre-checks

Before scanning for corpses, several guard conditions are evaluated in order:

CheckWhat It DoesOutcome If True
LNS.PauseLooting User paused looting via /lns pause Return immediately
LNS.SafeZones[LNS.Zone] Current zone is in the safe-zone list (looting disabled) Return immediately
Invisibility (types 1, 2, 3) Checks mq.TLO.Me.Invis() for normal, undead, and animal invis Return to avoid breaking invis
LNS.InCombat() Checks CombatState and XTarget list for aggressive mobs Return unless CombatLooting is enabled
MobsTooClose Counts hostile NPCs within MobsTooClose + CorpseRadius range Return unless CombatLooting is enabled
Dead + Corpse count == 0 No NPC or player corpses in range Return immediately
Loot window already open Another loot operation is in progress Return immediately to avoid conflicts
Casting a spell (non-Bard) Moving would interrupt casting Return immediately

Player Corpse Handling

In EQ, when your character dies you leave behind a corpse containing all your gear. LootNScoot checks for your own corpses using a SpawnCount TLO filter: 

pccorpse <YourName>'s radius <CorpseRadius> zradius <CorpseZRadius>

(This is a spawn search filter — a "spawn" is any entity in the EQ game world, including NPCs, players, and corpses. The TLO query returns a count of matching spawns within the specified radius.)

Three settings control how player corpses are handled:

  • LootMyCorpse (default: false) — If true, your own corpse is targeted, pulled with /corpse, and looted with /lootall.
  • IgnoreMyNearCorpses (default: false) — If true, skip your own corpse entirely and proceed to NPC corpses. Useful on servers where you respawn with all your gear.
  • If both are false and your corpse is nearby, looting pauses entirely until the corpse is gone (e.g., resurrected by a cleric).

NPC Corpse Scanning

NPC corpses are found using MQ's SpawnCount TLO with a filter string: 

npccorpse radius <CorpseRadius> zradius <CorpseZRadius>

CorpseRadius (default 100) is the horizontal distance in game units, and CorpseZRadius (default 50) is the vertical distance. Each corpse found is checked against:

  • CheckCorpseOnce — If enabled (default true), corpses that have already been looted (tracked in LNS.lootedCorpses) are skipped.
  • checkLockedCorpse() — Corpses that triggered a "can't loot" event are temporarily locked for LootCheckDelay seconds before retrying.
  • Path length — The navigation path length must be less than CorpseRadius to avoid chasing corpses through long winding tunnels.

Directed Mode Gate

If LNS.Mode == 'directed' (the script was started as a library by another script), looting only proceeds when LNS.LootNow has been set to true by the directing script. Otherwise the function returns early.

Navigation: navToID()

LNS.navToID(spawnID) moves your character to a corpse using the MQ2Nav plugin, which provides an in-game GPS/pathfinding system. The function:

  1. Checks if you are already within 10 units of the target (if so, returns immediately).
  2. Closes the loot window if one is open (to avoid movement conflicts).
  3. Issues /nav id <spawnID> dist=10 log=off to begin pathfinding.
  4. Waits in a loop, checking mq.TLO.Navigation.Active() every 50ms.
  5. Times out after NavTimeout seconds (default 5) to prevent getting stuck.
  6. Waits up to 100ms for movement to stop (mq.TLO.Me.Moving()).
Note: MQ2Nav requires a mesh file for the current zone. If no mesh exists, navigation commands silently fail and the character will not move to the corpse.

Processing a Corpse: lootCorpse()

LNS.lootCorpse(corpseID) handles everything from opening the loot window to picking up each item. Here is the sequence:

Opening the Loot Window

The function first clears anything on the cursor via checkCursor(), waits for the character to stop moving and casting, then attempts to open the loot window: 

for i = 1, 3 do
    mq.cmdf('/loot')
    mq.delay(1000, function()
        return mq.TLO.Window('LootWnd').Open()
    end)
    if mq.TLO.Window('LootWnd').Open() then break end
end

mq.cmdf() executes an in-game slash command (here, /loot to open the loot window on the targeted corpse). mq.delay(1000, fn) pauses script execution for up to 1000 milliseconds, but returns early if the callback function returns true (the window opened). It tries up to 3 times, waiting 1 second each attempt. After the loop, it processes the CantLoot event and waits one more second. If the window still is not open and no CantLoot event fired, it tries one more time (a 4th attempt). If another player is looting the same corpse (CantLoot event), the corpse is added to cantLootList with a timestamp so it is temporarily skipped.

Item Scanning Loop (Phase 1: Evaluate)

The looting process uses a two-phase approach: first scan all items to evaluate rules, then loot them one at a time with rechecks. This separation lets the script classify everything up front before committing to any pickups.

Once the loot window is open, the function iterates through every item in the corpse (mq.TLO.Corpse.Items() returns the count). For each item:

  1. Calls LNS.getRule(corpseItem, 'loot', slotIndex) to determine the action (Keep, Sell, Bank, Destroy, Ignore, Ask, Quest, etc.).
  2. Records the item data to the Items.db via addToItemDB().
  3. Separates items into two lists: lore items and non-lore items.

Why Lore and Non-Lore Are Separated

In EQ, Lore items are unique — you can only carry one at a time. If you already have a lore item and try to pick up a second copy, the game rejects it. By processing non-lore items first, LootNScoot avoids a situation where a failed lore pickup disrupts the loot window state and prevents looting of subsequent items. Lore items are then processed in a second pass, after all safe pickups are complete. The rule system also checks for lore duplicates early in its decision tree to skip them before they even reach this point.

Item Looting Loop

After scanning, both lists are iterated in order. For each actionable item (not Ignore, Ask, or NULL), lootItem() is called. Items marked Ignore or Ask are logged to history. Skipped items (Ask) are collected and reported at the end via reportSkippedItems().

Quest and Lore Rechecks at Loot Time

Before each individual lootItem() call, the looting engine rechecks quest item counts and lore status. This is necessary because the initial scan phase evaluates all items on the corpse at once, but picking up items changes your inventory between pickups. Without rechecking, over-looting can occur.

Quest items: If a corpse contains 5 copies of a quest item and you only need 1 more to reach your qKeep target, the scan phase would mark all 5 as "Quest" (lootable). The recheck before each pickup compares the current FindItemCount + FindItemBankCount against the qKeep target. After picking up the first copy, the count is met, so the remaining 4 are skipped and recorded to history as "Left".

Lore items: If a corpse somehow contains multiple copies of a lore item, the first pickup succeeds. The recheck before the second pickup detects that FindItemCount + FindItemBankCount > 0, so the duplicate is skipped. Without this recheck, attempting to loot a second lore item would be rejected by the game client and could disrupt the loot window state for remaining items.

After all items are processed, the loot window is closed by clicking the Done button: 

mq.cmdf('/nomodkey /notify LootWnd LW_DoneButton leftmouseup')

Picking Up Items: lootItem()

LNS.lootItem(mq_item, index, doWhat, button, qKeep, cantWear) handles the actual pickup of a single item from an open loot window.

The /itemnotify Command

MQ provides /itemnotify to simulate mouse clicks on UI elements. To loot slot number index: 

mq.cmdf('/nomodkey /shift /itemnotify loot%s %s', index, button)

The /nomodkey prefix ensures no modifier keys interfere, and /shift is used to trigger the loot action.

NoDrop Confirmation Dialog

In EQ, NoDrop (also called "No Trade") items are permanently bound to the character who picks them up — they cannot be traded, sold on the bazaar, or given away. Because this is irreversible, the game shows a confirmation dialog asking if you really want to loot them. LootNScoot automatically clicks "Yes" on this dialog when the LootNoDrop setting is enabled: 

if mq.TLO.Window('ConfirmationDialogBox').Open() then
    mq.cmdf('/nomodkey /notify ConfirmationDialogBox Yes_Button leftmouseup')
end

Destroy Handling

If the action is "Destroy", the item is first looted to cursor, then immediately destroyed with /destroy. This removes the item from the game permanently. The DoDestroy setting must be enabled for destroy actions to execute; otherwise "Destroy" is treated as "Ignore".

Destruction is permanent. There is no undo. Items destroyed by /destroy are gone forever. The DoDestroy setting defaults to false as a safety measure. Only enable it after you are confident your rules are set correctly.

Cursor Management: checkCursor()

After looting, LNS.checkCursor() ensures nothing is stuck on the cursor. It loops while mq.TLO.Cursor() is not nil, issuing /autoinv to place cursor items into inventory. If inventory is completely full, it logs a warning and returns.

Reporting

If ReportLoot is enabled, looted items are announced to the configured LootChannel (default "dgt", which is DanNet group tell — DanNet is an older MQ communication plugin; LootNScoot uses the newer actor system for its own coordination, but the loot channel can still use DanNet commands). Quest items include a count display showing progress (e.g., "3 of 10").

Foraging

EQ Rangers and Druids can forage items from the ground. When the game's "forage" event fires, LNS.eventForage() is triggered. It processes items that appear on cursor:

  1. Waits up to 1 second for the foraged item to appear on cursor.
  2. Calls getRule(cursorItem, 'forage', 0) to evaluate the item.
  3. If the rule says to keep it (and inventory has space), issues /autoinv.
  4. If the rule says ignore (or quest count is met), issues /destroy.
  5. Loops until the cursor is clear (multiple items can queue up).

The LootForage setting (default true) must be enabled. The LootForageSpam setting controls whether forage actions are echoed to the MQ console.

Processing Inventory

LNS.processItems(action) is the state machine that handles selling, banking, tributing, destroying, and buying. It iterates through all 10 inventory bag slots (skipping IgnoreBagSlot if set), examines every item inside each bag, and performs the requested action on items whose rule matches.

The function temporarily disables AlwaysEval during processing to prevent re-evaluation of rules mid-operation, restoring it afterward.

Selling

Triggered by /lns sellstuff or LNS.processItems('Sell').

  1. goToVendor() — Checks that you have a vendor targeted. If you are more than 15 units away, calls navToID() to walk to them.
  2. openVendor() — Right-clicks the target to open the Merchant window. Waits up to 5 seconds for MerchantWnd to appear.
  3. SellToVendor() — For each item marked "Sell", uses /itemnotify to select the item in inventory, waits for the merchant to recognize it, then clicks the Sell button with shift held (to sell the full stack).

The NEVER_SELL list is a hardcoded table of items that must never be sold regardless of rules: 

local NEVER_SELL = {
    ['Diamond Coin'] = true,
    ['Celestial Crest'] = true,
    ['Gold Coin'] = true,
    ['Taelosian Symbols'] = true,
    ['Planar Symbols'] = true,
}

After selling, profit is calculated by comparing platinum before and after: (mq.TLO.Me.Cash() - myCoins) / 1000. The result is reported to the loot channel.

Banking

Triggered by /lns bankstuff. Requires targeting a Banker NPC.

  1. openBanker() — Right-clicks the target to open BigBankWnd.
  2. bankItem() — Picks up each Bank-marked item with /shift /itemnotify, then clicks the Auto button in the bank window to deposit it. If the bank is full, the item is returned to inventory via /autoinventory.

The BankTradeskills setting (default false) causes the processItem helper to treat any item flagged as a tradeskill material as "Bank" even if it has no explicit rule. The /lns tsbank command bulk-marks all tradeskill items currently in your bags as "Bank".

Tributing

Triggered by /lns tributestuff. Requires targeting a Tribute Master NPC.

  1. openTribMaster() — Right-clicks the target to open TributeMasterWnd.
  2. TributeToVendor() — Selects each Tribute-marked item, waits for the tribute value to appear and the Donate button to enable, then clicks Donate. Handles the quantity dialog if it appears.

Items in the NEVER_SELL list are also protected from tributing. After tributing, the window is closed and bags are collapsed.

Destroying

Triggered by /lns cleanup.

LNS.DestroyItem(item, bag, slot) picks up the item to cursor with /shift /itemnotify, waits for it to appear on cursor, then issues /destroy. Items in NEVER_SELL are protected.

Warning: The DoDestroy setting (default false) must be explicitly enabled for destroy to work. When disabled, items marked "Destroy" are treated as "Ignore" and left on corpses. This is a safety measure to prevent accidental item loss.

Restocking

LNS.RestockItems() buys items listed in the BuyItemsTable from a targeted merchant. For each item in the table:

  1. Searches the merchant's item list for the item name.
  2. Calculates how many are needed: qty - mq.TLO.FindItemCount(itemName)().
  3. Selects the item row, clicks Buy, enters the quantity in the QuantityWnd dialog.
  4. Loops if the stack size is smaller than the desired quantity (buys multiple stacks).
  5. Retries up to 5 times if item selection fails.

Restocking happens automatically after selling when AutoRestock is enabled (default true), or on demand via /lns restock.

The BuyItemsTable format is {[itemName] = quantity}. Items are added via /lns buy "Item Name" 5 or through the UI.

Looting Sequence Flow

lootMobs(limit)
    |
    +-- Pre-checks (invis, pause, safe zone, combat, bags)
    |       |
    |       +-- FAIL --> return false
    |
    +-- Scan for player corpses
    |       |
    |       +-- LootMyCorpse? --> /corpse + /lootall
    |       +-- IgnoreMyNearCorpses? --> skip, continue
    |       +-- Neither? --> pause looting, return
    |
    +-- SpawnCount('npccorpse radius R zradius Z')
    |       |
    |       +-- Build corpseList (filter looted, locked, path length)
    |
    +-- Directed mode gate (skip if not LootNow)
    |
    +-- For each corpse in corpseList (up to limit):
        |
        +-- navToID(corpseID)
        |       |
        |       +-- /nav id <ID> dist=10
        |       +-- Wait up to NavTimeout seconds
        |
        +-- lootCorpse(corpseID)
        |       |
        |       +-- /loot (3 attempts, 1s each)
        |       +-- Scan items: getRule() for each
        |       +-- Separate lore vs non-lore
        |       +-- Loot non-lore first, then lore
        |       |       |
        |       |       +-- lootItem(item, index, action, ...)
        |       |       +-- If item still on corpse, retry lootItem once
        |       |               |
        |       |               +-- /itemnotify loot<N> leftmouseup
        |       |               +-- Handle NoDrop confirmation
        |       |               +-- Handle Destroy (/destroy)
        |       |               +-- checkCursor() --> /autoinv
        |       |               +-- Log to history
        |       |
        |       +-- Close loot window (Done button)
        |       +-- reportSkippedItems()
        |
        +-- Record in lootedCorpses
        +-- Send actor message with looted items
Tip: The MaxCorpsesPerCycle setting (default 5) controls how many corpses are processed in a single lootMobs() call. Lower values reduce the time your character spends running between corpses, which is helpful in dangerous areas. Higher values are more efficient when the area is safe.
